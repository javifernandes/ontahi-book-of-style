# Diagrams

Diagrams should be sparse and conceptual.

Use diagrams to reveal categories, not to impress with complexity.

Preferred diagram types:

- entity relationship sketches
- event loops
- durable operation layers
- runtime placement maps
- authority boundaries
- synchronization paths

Avoid overly literal infrastructure diagrams unless necessary.
