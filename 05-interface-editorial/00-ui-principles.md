# UI Principles

1. Clarity before density.
2. Meaning before interaction.
3. Whitespace before decoration.
4. Editorial composition before dashboards.
5. Calm before excitement.
6. Organic rhythm before rigid grids.
7. Code examples should feel like notebooks.
8. Dark mode belongs mainly to code and editor surfaces.

This part defines an \concept{interface language}, not a component catalog.
