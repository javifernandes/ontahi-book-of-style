# AGENTS Instructions for `ontahi-book-of-style`

Use this repository as style and context for Ontahí artifacts.

Guidelines:

1. Keep the book lightweight and useful for humans and agents.
2. Preserve numbered folders as parts and numbered Markdown files as chapters.
3. Prefer small, precise additions over broad expansion.
4. Do not invent public architecture claims that are not supported by the current implementation.
5. Treat tasks as the current durable execution surface and durable operations as the architectural exposure of that behavior.
6. Avoid using workflow as a core term for the current model; reserve it for a possible future dynamic orchestration layer.
7. Keep Ontahí visual language editorial, botanical, organic, calm, and technically serious.
8. Avoid generic SaaS, AI-hype, crypto, cyberpunk, neon, glassmorphism, and dashboard-heavy aesthetics.
9. When adding assets, prefer small composable pieces that can combine from micro to macro, like a biological system.
10. Keep `bookops.manifest.json` aligned with the folder structure when adding, removing, or renaming parts.
